## Codebase Patterns
- UI typecheck: run `yarn typecheck` from ui/ (no root package.json). It runs `vue-tsc --noEmit`.
- Lit components registered via `./components/lit/index` import in ui/src/main.ts.
- Pinia uses Options API `defineStore` style for this project.
- IndexedDB via Dexie: `ui/src/db.ts` exports `db` (SurfsAppDB) with `favourites` (&beachId) and `settings` (&key) tables. Import as `import { db } from '../db'` from stores/components.
- New lucide icons: import in `ui/src/components/lit/surf-icon.ts`, add to ICONS map keyed by kebab-case name. Lucide exports `Map` and `List` as PascalCase.
- Route names are referenced via `router.push({ name: 'xxx' })`. Existing names: `beach-detail`. Phase 13 adds `beach-map`.
- FAB / overlay pattern: position fixed, use env(safe-area-inset-*) on offsets, `--color-ocean-800` is the brand accent.
- MapLibre: import `'maplibre-gl/dist/maplibre-gl.css'` in any view that mounts a map (BeachMap.vue does NOT — it relies on the importer to provide CSS). Without it controls/markers render unstyled.
- Beach coords on `Beach` type are `{ lat, lon }`; MapLibre expects `[lon, lat]` order — easy to flip by mistake.
- MapLibre paint expressions cannot consume CSS custom properties — read them via `getComputedStyle(document.documentElement).getPropertyValue('--color-...')` at map init and pass the resolved hex into a `match` expression on a feature property.
- Demotiles style glyphs include `Noto Sans Regular` — use it as the `text-font` value for symbol layers there. Other demotiles fonts are not guaranteed.
- maplibre-gl v5 `GeoJSONSource.getClusterExpansionZoom(clusterId)` returns a Promise<number> (not a callback). Cast `map.getSource(id)` to `maplibregl.GeoJSONSource` to access cluster helpers.
- Cluster + non-cluster filters: `['has', 'point_count']` for clusters, `['!', ['has', 'point_count']]` for individual features. Add layers in render order — clusters and individual circles are mutually exclusive via the filters, so order matters only for stacking with future overlays.
- Persist UI state in `db.settings` (key/value table) with typed validators when reading. Narrow `unknown` values through type guards like `isValidCenter` / `isValidZoom` — do NOT cast with `as`. Match the pattern used by `hydrateSettings` in beachStore.ts.
- Debounce high-frequency map event writes (`moveend` fires continuously during panning) with a `setTimeout` handle stored at module scope; clear it in `onBeforeUnmount` to avoid writes-after-unmount.
- Auto-dismissing transient notices (e.g. geolocation error) follow the same module-scoped `setTimeout` handle pattern: clear-then-restart on each show, and clear in `onBeforeUnmount` so a 4s timer can't fire on a torn-down view.
- `surf-button` exposes its inner button element as `part="button"` — target it from outside with `surf-button::part(button) { ... }` to override width/size. Host is `display: inline-flex` by default; for full-width usage also set host `display: block; width: 100%`.
- MapLibre layer click handlers receive features in `e.features` (typed as `MapGeoJSONFeature[] | undefined`); `properties` is `Record<string, any>` so narrow with `typeof`.
- Online/offline state: use `useOnlineStatus()` from `ui/src/composables/useOnlineStatus.ts` — it returns a reactive `isOnline` ref and self-cleans via `onScopeDispose`. Don't poll `navigator.onLine`.
- MapLibre `addImage` requires the image be loaded. For inline SVG icons, build a `data:image/svg+xml` URL and load via `new Image()` + `onload` Promise BEFORE calling `addImage`. Load it in parallel with other async setup (`Promise.all` alongside `db.settings.get(...)`), then register inside the `map.on('load', ...)` callback.
- For source feature properties consumed by symbol-layer expressions like `['get', 'windDirection']`: OMIT the property entirely when null (don't set `key: null`). MapLibre's `['has', key]` filter returns true for null values, so keeping `key: null` won't exclude features. Build the properties object conditionally.
