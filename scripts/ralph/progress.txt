## Codebase Patterns
- UI uses Vite + Vue 3 + TypeScript, package manager is yarn
- UI typecheck: `cd ui && yarn typecheck`
- Vue custom element config in vite.config.ts: isCustomElement: (tag) => tag.startsWith('surf-')
- Lit Web Components use `declare` + constructor pattern for reactive properties (not class field initializers)
- Available Lit components: surf-card (clickable, padding), surf-badge (variant, size), surf-icon (name, size, color), surf-button, surf-bottom-sheet
- surf-icon available names: waves, heart, map-pin, wind, star, chevron-right, arrow-left, refresh-cw
- surf-badge variant values: 'very-good', 'good', 'maybe', 'poor', 'neutral' — these map directly to currentLabel from the API
- Design tokens: CSS custom properties on :root — --space-N, --font-size-X, --color-*, --font-weight-*
- SCSS $variables for breakpoints (not CSS custom props — can't use in @media)
- Surf score colors: --color-surf-very-good (#2d9e5f), --color-surf-good (#1a72c4), --color-surf-maybe (#d4a017), --color-surf-poor (#c0392b)
- Backend API at http://localhost:3000: GET /beaches, GET /beaches/:id, GET /beaches/:id/forecast, GET /health, POST /admin/fetch-forecasts
- GET /beaches response shape: [{ id, name, region, coords, skillLevel, tags, currentScore, currentLabel, lastUpdated }]
- GET /beaches/:id/forecast response: { beachId, fetchedAt, hourlyForecasts: [{ timestamp, rawData, surfScore, label, reasons, confidence }], dailySummaries: [{ date, bestWindowStart, bestWindowEnd, peakScore, overallLabel }] }
- rawData keys: swell_wave_height, swell_wave_period, swell_wave_direction, swell_wave_height_2, swell_wave_direction_2, wind_speed_10m, wind_direction_10m, wave_height
- Router at ui/src/router.ts — routes: /, /design, /components
- No Pinia installed yet — US-001 will add it
- ui/src/main.ts is the app entry point
- Vite serves ui/public/* at the site root; reference static assets as `/filename.ext` (not `/public/...`)
